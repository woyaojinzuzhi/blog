React-Native的期末考复习

RN特点

  • 学习一次，可以在任意地方使用，可以使用react构建移动端app，使用JS和React编写原生移动应用。
  • 不用WebView，拜托WebView的交互和性能问题
  • 封装原生控件有更好的触摸滚动体验和灵敏的手势识别
  • 支持热更新
  • Facebook在2015年9月公布的开源项目

三种APP开发模式：

  • Native APP
    基于本地（操作系统）运行的App，优点：速度快，性能高，用户体验好；缺点：无法跨平台，开发成本高，更新麻烦。
  • Web App
    基于移动设备的浏览器运行的App；优点：开发周期短，开发成本低，更新简单，维护轻松；缺点：用户体验不佳。
  • Hybrid App
    基于上两种发展出来的产物，结合Native App用户体验效果好和Web App的跨平台开发的优势。通常是基于第三方跨平台移动开发框架进行开发。继承了Web App实时更新、开发成本低等优点；通过应用商店区分移动操作系统，分发用户需安装使用的移动应用。使用方式和Native App一致。

环境搭建

简单流程、node、jdk、SDK（他们的作用）

安装Node，安装JDK，安装SDK（在这之后有配置gradle，RN最后打包需要gradle）、安装模拟器、创建-运行项目。

  • jdk，是Java的核心，java运行的环境，Java工具和Java基础的类库—-RN应该是由Java写的，所以要有运行的环境。
  • sdk，是软件开发工具包，用于创建应用软件的工具集合。RN应该是集成了sdk，由相应的特殊功能，一个产品要想实现某些特定功能如消息推送，便可找到相关的第三方sdk，不用再重新开发。
  • node是RN的运行环境，同时使用node时，可以利用npm命令启动，也可以通过npm下载、安装项目，下载第三方资源包。

状态

redux

Redux是JavaScript状态容器，提供可预测化的状态管理。目的：随着JavaScript单页应用开发日趋复杂，JavaScript需要管理比任何时候都要多的state（状态），管理不断变化的state非常困难；核心概念：Action、Reducer、Store

Action

  • Action是把数据从应用传到store的有效载荷
  • 是store数据的唯一来源
  • 一般会通过store.dispatch()将对action传到store
  • Action本质上是JavaScript的普通对象
  • Action内必须使用一个字符串类型的type字段来表示将要执行的动作
  • 一般type会被定义成字符串常量
  • 当应用规模越来越大时，建议使用单独的模块或文件来存放action

Reducer

  • 指定了应用状态的变化如何响应action并发送到store
    reducer就是一个纯函数，接收旧的state和action，返回新的state
    (previousState,action) => newState
  • 禁止在Reducer里做这些操作
  • 修改传入参数
  • 执行有副作用的操作，如API请求和路由跳转
  • 调用非纯函数，如Date.now() 或 Math.random()

Store

  • Store是把Action和Reducer联系到一起的对象
  • Redux应用只有一个单一的store
  • 作用：
    • 维持应用的state；
    • 提供getState()方法获取state；
    • 提供dispatch(action)方法更新state；
    • 通过subscribe(listener)注册监听器
    • 通过subscribe(listener)返回的函数注销监听器
